## 🔍 Project Overview

Next word prediction is a foundational NLP task that powers autocomplete systems, smart keyboards, and language models. This project builds an **end-to-end deep learning language modeling pipeline** that:

1. Loads and tokenizes Shakespeare's *Hamlet* using NLTK's Gutenberg corpus
2. Generates n-gram input sequences from the text
3. Pads all sequences to a uniform length of 13 tokens
4. Trains a **stacked LSTM** with an Embedding layer and Dropout regularization
5. Predicts the single most probable next word for any input phrase in real time

---

## 🎯 Problem Statement

**Multi-class Word Prediction:** Given a sequence of words as input, predict the single most likely next word from a vocabulary of **4,818 unique words** extracted from Hamlet.

The model is trained as a language model — for every n-gram subsequence in the text, the last word is the label and all preceding words are the input features.

---

## 📊 Dataset

**Source:** Shakespeare's *Hamlet* — loaded via `nltk.corpus.gutenberg` (`shakespeare-hamlet.txt`) and saved locally as `hamlet.txt`

| Property | Details |
|---|---|
| Text Source | Shakespeare's Hamlet (NLTK Gutenberg corpus) |
| Total Unique Words (Vocabulary) | **4,818** |
| Total Input Sequences Generated | **25,732** |
| Max Sequence Length | **13 tokens** (longest n-gram in text) |
| Train / Test Split | 80% / 20% |
| Input Shape | `(25,732, 13)` after padding |
| Output Shape | `(25,732, 4,818)` one-hot encoded |

Sequences are generated by sliding over every line of the text — for each line, all n-gram subsequences from length 2 up to the full line length are created, giving the model a wide range of context lengths to learn from.

## 🏗 Project Architecture

```
Shakespeare's Hamlet Text (hamlet.txt)
              │
              ▼
   Text Preprocessing
  ├─ Lowercase the entire text
  ├─ Fit Keras Tokenizer → vocabulary of 4,818 words
  ├─ Generate all n-gram subsequences from each line
  ├─ Pad sequences to max_len = 13 (pre-padding with zeros)
  ├─ Split: X = all tokens except last, y = last token
  └─ One-hot encode y → shape (25732, 4818)
              │
              ▼
    Stacked LSTM Model (TensorFlow/Keras)
  ├─ Embedding Layer  → (4818 vocab, 100 dims, seq_len=13)
  ├─ LSTM Layer 1     → 150 units, return_sequences=True
  ├─ Dropout          → rate = 0.2
  ├─ LSTM Layer 2     → 100 units
  └─ Dense Output     → 4818 units, Softmax activation
              │
              ▼
  argmax(prediction) → Predicted Word Index
              │
              ▼
  Reverse lookup in tokenizer.word_index → Predicted Word
              │
              ▼
  Streamlit Web App → Real-time next word display
```

## 🔄 Data Preprocessing Pipeline

**Step 1 — Load & Lowercase Text**
```python
with open('hamlet.txt', 'r') as file:
    text = file.read().lower()
```

**Step 2 — Tokenize**
```python
tokenizer = Tokenizer()
tokenizer.fit_on_texts([text])
total_words = len(tokenizer.word_index) + 1  # → 4818
```

**Step 3 — Generate N-gram Sequences**
```python
# For each line, generate all prefix n-grams
# e.g., "to be or not" → [to,be], [to,be,or], [to,be,or,not]
for line in text.split('\n'):
    token_list = tokenizer.texts_to_sequences([line])[0]
    for i in range(1, len(token_list)):
        input_sequences.append(token_list[:i+1])
# Total: 25,732 sequences
```

**Step 4 — Pad Sequences**
```python
max_sequence_len = 14   # longest n-gram = 13 tokens + 1
input_sequences = pad_sequences(input_sequences, maxlen=max_sequence_len, padding='pre')
# Shape: (25732, 14)
```

**Step 5 — Create X and One-Hot y**
```python
X = input_sequences[:, :-1]          # shape: (25732, 13)
y = to_categorical(input_sequences[:, -1], num_classes=total_words)
# shape: (25732, 4818)
```

**Step 6 — Inference (app.py)**
```python
def predict_next_word(model, tokenizer, text, max_sequence_len):
    token_list = tokenizer.texts_to_sequences([text])[0]
    if len(token_list) >= max_sequence_len:
        token_list = token_list[-(max_sequence_len - 1):]
    token_list = pad_sequences([token_list], maxlen=max_sequence_len-1, padding='pre')
    predicted = model.predict(token_list, verbose=0)
    predicted_word_index = np.argmax(predicted, axis=1)
    for word, index in tokenizer.word_index.items():
        if index == predicted_word_index:
            return word
```

|---|---|---|---|---|
| 1 | 3.27% | 6.9113 | 3.28% | 6.7302 |
| 10 | 9.06% | 5.3779 | 6.24% | 7.1882 |
| 20 | 15.26% | 4.2528 | 6.63% | 8.3683 |
| 30 | 26.34% | 3.5197 | 5.63% | 9.5340 |
| 40 | 35.00% | 3.0295 | 5.60% | 10.4508 |
| 50 | 41.53% | 2.6539 | 5.52% | 11.1871 |

**Important Note on Results:** The growing gap between training and validation accuracy is expected behavior for a character-level language model trained on a small, domain-specific corpus like Hamlet. The model learns the stylistic patterns and word co-occurrences specific to Shakespearean English. A larger corpus or a pre-trained embedding (e.g. GloVe) would improve generalization significantly — this is the key motivation behind modern large language models.

## 💡 Key Learnings

- **Language Modeling with LSTM:** Learned how stacked LSTMs capture long-range sequential dependencies in text far better than Simple RNNs — the first LSTM with `return_sequences=True` passes the full sequence of hidden states to the second LSTM, which compresses it into a single context vector
- **N-gram Sequence Generation:** Built training data by generating all possible prefix n-grams from each line of text — this technique maximizes training signal from a limited corpus
- **Word Embeddings:** Used a trainable `Embedding` layer (100-dim) to learn dense word representations from scratch, capturing semantic relationships between Shakespearean vocabulary
- **Dropout Regularization:** Applied `Dropout(0.2)` between LSTM layers to reduce co-adaptation of neurons and improve generalization
- **Categorical Crossentropy for Multi-class NLP:** Framed next word prediction as a multi-class classification problem over the full 4,818-word vocabulary with `to_categorical` one-hot encoding and Softmax output
- **Tokenizer Serialization:** Saved the fitted Keras `Tokenizer` with Pickle to ensure identical vocabulary mapping between training and inference — a critical production practice
- **Streamlit Deployment:** Built and deployed a real-time inference app that loads the `.h5` model and `.pickle` tokenizer, applies the same preprocessing, and returns predictions with zero latency
